Welcome

Roles: Admin (maintenance, reports, transactions) and User (reports, transactions). Backend uses Flask + SQLite.

Admin Login

Full access

Passwords are hidden on input. Admin can reach maintenance, reports, and transactions.

User Login

Reports + Transactions

Users cannot access maintenance screens.

Maintenance (Admin)

Seed data for reports and transactions.

  • Add/Update memberships (6m default; 1y/2y options)
  • Add/Update books or movies (book default; all fields required)
  • Add/Update users (name required; new user default)
Go to Maintenance

Reports

Accessible by admin and users.

  • Active issues
  • Master list of memberships
  • Master list of movies
  • Master list of books
  • Overdue returns
  • Pending issue requests
View Reports

Transactions

Issue, return, and pay fines with in-form validation.

  • Check availability (radio select a book row)
  • Issue book (title required; author auto; return date ≤ +15d; remarks optional)
  • Return book (title, serial no.; author/issue date auto; editable return date)
  • Fine payment (confirm after fine paid when due)
Start Transaction

Notes for Backend

  • SQLite persistence (members, users, books/movies, issues, returns, fines).
  • Enforce validation: book availability; required fields; return date rules; fine paid before completion.
  • Default selections: book vs movie → book; new user; membership → 6 months; membership extension → 6 months; return date prefilled to issue date + 15 days.
  • Chart link on every page (header uses the flow chart image).
